  <files>backend/pipeline/jobs/extract_aspects.py</files>
  <action>
    Create `backend/pipeline/jobs/extract_aspects.py`.

    **Module docstring:**
    ```
    Tier 2 LLM aspect extraction job for VibeCheck Phase 8.

    Routes posts with low Tier 1 confidence (< 0.6) to the configured LLM provider
    for per-entity aspect-level sentiment extraction. Stores results in AspectSentiment.

    Routing criteria (ALL must be true):
    - Post.sentiment_score < 0.6 (any label — Positive, Negative, Neutral)
    - Post.sentiment_label IS NOT NULL (scored by Tier 1)
    - No existing AspectSentiment rows for this post (idempotent)

    Per-run volume cap: LLM_MAX_CALLS_PER_RUN env var (default 100).
    Returns stats dict compatible with wrapped_pipeline_execution() audit logging.
    ```

    **Imports:**
    ```python
    import logging
    import os
    from sqlalchemy.ext.asyncio import AsyncSession
    from sqlalchemy import select, and_, exists, insert
    from db.models import Post, PostEntityMention, AspectSentiment
    from pipeline.services.llm_provider import get_llm_provider
    from utils.constants import VALID_ASPECTS
    ```

    **Constants:**
    ```python
    logger = logging.getLogger(__name__)
    CONFIDENCE_THRESHOLD = 0.6  # Posts below this are routed to Tier 2
    ```

    **run_extract_aspects(session: AsyncSession) -> dict:**

    1. **Initialize stats:** `{"routed": 0, "extracted": 0, "errors": 0, "skipped": 0}`

    2. **Load volume cap:** `max_calls = int(os.getenv("LLM_MAX_CALLS_PER_RUN", "100"))`

    3. **Routing query (idempotent):**
       ```python
       has_aspects = exists(
           select(AspectSentiment.id).where(AspectSentiment.post_id == Post.id)
       )
       query = select(Post.id, Post.title, Post.body).where(
           and_(
               Post.sentiment_score < CONFIDENCE_THRESHOLD,
               Post.sentiment_label.isnot(None),
               ~has_aspects,
           )
       ).order_by(Post.id).limit(max_calls)
       ```

    4. **Execute query:** `result = await session.execute(query)` → `posts_to_route = result.all()`

    5. **Early return if empty:** log and return stats

    6. **Log count:** `logger.info("extract_aspects: routing %d low-confidence posts", len(posts_to_route))`

    7. **Instantiate LLM provider once:** `provider = get_llm_provider()`

    8. **For each post:**
       a. `stats["routed"] += 1`
       b. Compose text: `text = ((post.title or "") + " " + (post.body or "")).strip()[:3000]`
       c. **Lookup entity names from PostEntityMention:**
          ```python
          mention_query = select(PostEntityMention.entity_id).where(
              PostEntityMention.post_id == post.id
          )
          mention_result = await session.execute(mention_query)
          entity_ids = [row.entity_id for row in mention_result.all()]
          ```
          Then lookup entity names:
          ```python
          from db.models import Entity
          entity_query = select(Entity.id, Entity.name).where(Entity.id.in_(entity_ids))
          entity_result = await session.execute(entity_query)
          entity_map = {row.name: row.id for row in entity_result.all()}  # name → entity_id
          entity_names = list(entity_map.keys())
          ```
          If no entities found for post, log warning and `stats["skipped"] += 1`, continue.

       d. **Call LLM provider with retry:**
          ```python
          try:
              aspects_by_entity = await provider.extract_aspects(text, entity_names)
          except Exception as exc:
              logger.error("extract_aspects: LLM failed for post %s: %s", post.id, exc)
              stats["errors"] += 1
              continue  # Skip this post — will retry next run
          ```

       e. **Write AspectSentiment rows:**
          For each entity_name in aspects_by_entity:
          - Lookup entity_id from entity_map — if not found (hallucinated entity), log and skip
          - For each aspect_name in VALID_ASPECTS:
            - score = float(aspects_by_entity[entity_name].get(aspect_name, 0.0))
            - Clamp to [-1.0, 1.0] as safety: `score = max(-1.0, min(1.0, score))`
            - Create AspectSentiment row with (post_id=post.id, entity_id=entity_id, aspect=aspect_name, score=score)
          - Use `insert(AspectSentiment).values(...)` with ON CONFLICT DO NOTHING for safety:
            ```python
            from sqlalchemy.dialects.postgresql import insert as pg_insert
            stmt = pg_insert(AspectSentiment).values(
                post_id=post.id, entity_id=entity_id, aspect=aspect_name, score=score
            ).on_conflict_do_nothing(
                index_elements=["post_id", "entity_id", "aspect"]
            )
            await session.execute(stmt)
            ```

       f. `await session.commit()` after each post's aspects are written (keeps transactions small)
       g. `stats["extracted"] += 1`

    9. **Log summary:** `logger.info("extract_aspects: complete — routed=%d, extracted=%d, errors=%d, skipped=%d", ...)`

    10. **Return stats**

    **Key constraints from CONTEXT.md (locked):**
    - Confidence threshold = 0.6 (strict, all labels eligible)
    - Idempotent: NOT EXISTS check on AspectSentiment prevents reprocessing
    - Retry logic handled inside llm_provider.py — job just catches final exception
    - After exhausted retries, log error, skip post (continue loop), do NOT crash job
    - No fallback to secondary provider

    **From RESEARCH.md anti-patterns:**
    - Do NOT broadcast aspects to all entities in post — link each to specific entity_id
    - Do NOT skip aspect storage if score=0.0 — store all 7 aspects (simplifies aggregation)
    - Do NOT let one failing post crash the entire job run

    **Note on TimescaleDB FK constraint:** PostEntityMention.post_id has no DB-level FK to posts.id (TimescaleDB hypertable restriction). Application-level lookup via SELECT is the correct pattern (same as existing code in models.py).
