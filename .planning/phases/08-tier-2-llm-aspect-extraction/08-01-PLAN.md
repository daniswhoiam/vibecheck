    **test_aspect_extraction.py** cases:
    - Post with sentiment_score=0.5 (< 0.6) and no existing AspectSentiment rows is routed to Tier 2
    - Post with sentiment_score=0.8 (>= 0.6) is NOT routed to Tier 2
    - Post with sentiment_score=0.5 that already has AspectSentiment rows is NOT processed again (idempotent)
    - Post with sentiment_label=None is NOT routed (not yet scored by Tier 1)
    - Successful extraction writes AspectSentiment rows with (post_id, entity_id, aspect, score) for each entity
    - All 7 aspects stored per entity in routed post (even if some are neutral 0.0)
    - LLM entity name matched to PostEntityMention by name lookup; unmatched names skipped
    - Stats dict returned: {routed, extracted, errors}

    **test_aspect_api.py** cases:
    - GET /entities/{id}/aspects?window=7d returns aspects with mean and count per aspect
    - GET /entities/{id}/aspects?window=30d returns aspects computed over 30-day window
    - GET /entities/{id}/aspects?window=90d returns aspects computed over 90-day window
    - GET /entities/{id}/aspects?window=7d&source=hn filters to only HN-sourced aspects
    - GET /entities/{id}/aspects returns 404 for unknown entity_id
    - Response includes all 7 aspects (even if count=0) or only those with data (TBD — document assumption)
