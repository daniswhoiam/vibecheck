  <files>backend/pipeline/services/llm_provider.py</files>
  <action>
    Create `backend/pipeline/services/llm_provider.py` with the following components:

    **Imports:** abc, asyncio, json, logging, os, typing. Lazy imports for groq and openai inside __init__ methods (avoids import errors when provider is not selected).

    **Pydantic schemas (validate LLM output — enforce score bounds):**
    - `AspectScoresSchema(BaseModel)`: one field per VALID_ASPECT with `Field(default=0.0, ge=-1.0, le=1.0)`. Use `model_config = ConfigDict(extra='ignore')` to silently drop unknown aspects the LLM may hallucinate.
    - `EntityAspectsSchema(BaseModel)`: name: str, aspects: AspectScoresSchema
    - `LLMResponseSchema(BaseModel)`: entities: list[EntityAspectsSchema]

    **SYSTEM_PROMPT constant:**
    ```python
    SYSTEM_PROMPT = """You are a sentiment analysis expert. Extract aspect-level sentiment scores for each AI tool or model mentioned in the post.

    Fixed aspects: performance, cost, reliability, ux, speed, code_quality, context_window
    Score range: -1.0 (very negative) to 1.0 (very positive). Use 0.0 for neutral/not mentioned.
    Output ONLY valid JSON, no markdown, no explanations."""
    ```

    **LLMProvider(ABC):**
    - Abstract method: `async def extract_aspects(self, post_text: str, entity_names: list[str]) -> dict[str, dict[str, float]]`
    - Docstring explains return format: `{"Claude": {"performance": 0.8, "cost": -0.2, ...}, ...}`

    **GroqProvider(LLMProvider):**
    - `__init__(self, api_key=None, model_id="llama-3.3-70b-versatile")`: lazy import `from groq import Groq`, store `self.client = Groq(api_key=api_key or os.getenv("GROQ_API_KEY"))`
    - `extract_aspects(self, post_text, entity_names)`:
      - Build user prompt: post_text (truncated to 3000 chars) + entity list + JSON schema hint
      - Define inner `_call_sync()` function that calls `self.client.chat.completions.create(...)` with `response_format={"type": "json_object"}`, `temperature=0.3`, `max_tokens=1000`
      - Decorate `_call_sync` with `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=8), reraise=True)` from tenacity
      - Call via `json_str = await asyncio.to_thread(_call_sync)` — NEVER call Groq synchronously in async context
      - Parse: `raw = json.loads(json_str)`, validate: `validated = LLMResponseSchema(**raw)`
      - Return: `{e.name: dict(e.aspects) for e in validated.entities}`
      - On ValidationError: log warning with truncated output, re-raise

    **OpenAIProvider(LLMProvider):**
    - `__init__(self, api_key=None, model_id="gpt-4o-mini")`: lazy import `from openai import AsyncOpenAI`, store async client
    - `extract_aspects(self, post_text, entity_names)`:
      - Same prompt structure as GroqProvider
      - Use `await self.client.chat.completions.create(...)` (AsyncOpenAI is natively async — no asyncio.to_thread needed)
      - Same retry decorator and Pydantic validation as GroqProvider
      - Return same dict format

    **get_llm_provider() factory:**
    ```python
    def get_llm_provider() -> LLMProvider:
        provider_name = os.getenv("LLM_PROVIDER", "groq").lower()
        model_id = os.getenv("LLM_MODEL", None)  # None = use provider default

        if provider_name == "groq":
            kwargs = {"model_id": model_id} if model_id else {}
            return GroqProvider(**kwargs)
        elif provider_name == "openai":
            kwargs = {"model_id": model_id} if model_id else {}
            return OpenAIProvider(**kwargs)
        else:
            raise ValueError(f"Unknown LLM_PROVIDER: '{provider_name}'. Supported: groq, openai")
    ```

    **Key constraints from CONTEXT.md (locked):**
    - asyncio.to_thread() for Groq (synchronous SDK) — never block event loop
    - tenacity retry: 3 attempts, exponential backoff 1s/2s/4s, reraise=True after exhaustion
    - score range enforced by Pydantic: ge=-1.0, le=1.0 on each aspect field
    - No fallback to secondary provider — raise after retries exhausted

    **Anti-patterns to avoid (from RESEARCH.md):**
    - Do NOT call groq.client.chat.completions.create() directly without asyncio.to_thread
    - Do NOT use raw string parsing or regex for JSON — use json.loads() + Pydantic
    - Do NOT define DeepInfra provider — out of scope for this phase
