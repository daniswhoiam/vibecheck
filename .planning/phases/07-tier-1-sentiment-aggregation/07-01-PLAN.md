    Modify `backend/db/models.py`:

    1. Add two columns to the `Post` class after the `created_at` field:
       ```python
       sentiment_label: Mapped[str | None] = mapped_column(String(20), nullable=True, index=True)
       sentiment_score: Mapped[float | None] = mapped_column(Float, nullable=True)
       ```
       Storage rationale: `sentiment_label` holds 'Positive'/'Negative'/'Neutral' strings (self-documenting in logs). `sentiment_score` holds confidence 0.0-1.0 from the zero-shot classifier. Both nullable so existing posts remain valid before scoring runs. Index on `sentiment_label` for efficient "unscored posts" queries (`WHERE sentiment_label IS NULL`).

    2. Add a new `SentimentRollup` class after `AspectSentiment`:
       ```python
       class SentimentRollup(Base):
           """Daily sentiment rollup with per-source breakdown.

           One row per (entity, day). Incrementally recomputed for the current UTC day
           on each aggregation job run. Past days are immutable once written.

           source_breakdown JSON structure:
               {"hn": {"mean": 0.4, "count": 12}, "reddit": {"mean": -0.1, "count": 8}, ...}
           """
           __tablename__ = "sentiment_rollup"

           id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
           entity_id: Mapped[int] = mapped_column(
               Integer, ForeignKey("entities.id", ondelete="CASCADE"), nullable=False, index=True
           )
           rollup_date: Mapped[datetime] = mapped_column(
               TIMESTAMP(timezone=True), nullable=False
           )
           sentiment_mean: Mapped[float | None] = mapped_column(Float, nullable=True)
           post_count: Mapped[int] = mapped_column(Integer, nullable=False, default=0)
           source_breakdown: Mapped[dict | None] = mapped_column(JSON, nullable=True)
           updated_at: Mapped[datetime] = mapped_column(
               TIMESTAMP(timezone=True), server_default=func.now(), onupdate=func.now(), nullable=False
           )
           created_at: Mapped[datetime] = mapped_column(
               TIMESTAMP(timezone=True), server_default=func.now(), nullable=False
           )

           entity: Mapped["Entity"] = relationship(back_populates="sentiment_rollups")

           __table_args__ = (
               UniqueConstraint('entity_id', 'rollup_date', name='uq_sentiment_rollup_entity_date'),
               Index('ix_sentiment_rollup_entity_date', 'entity_id', 'rollup_date'),
           )
       ```

       Use `JSON` type (not `JSONB`) for the `source_breakdown` column — SQLAlchemy's `JSON` type maps to `JSONB` on PostgreSQL automatically, which gives binary storage and indexing capability without needing to import `JSONB` separately.

    3. Add the `sentiment_rollups` relationship to the `Entity` class:
       ```python
       sentiment_rollups: Mapped[list["SentimentRollup"]] = relationship(
           back_populates="entity", cascade="all, delete-orphan"
       )
       ```

    Imports already in file: `Integer, Float, String, ForeignKey, Index, UniqueConstraint, func, TIMESTAMP` are already present. Add nothing extra unless `date` type is missing.

    Note on `rollup_date`: Store as `TIMESTAMP(timezone=True)` (midnight UTC of the day) rather than SQLAlchemy `Date` type. This avoids timezone edge cases in aggregation queries and keeps consistent with the rest of the schema.
